# Codex 和 Claude Code MCP+Hooks Benchmark / Codex and Claude Code MCP+Hooks Benchmark

本 benchmark 用两层证据证明 Codex 与 Claude Code 适配可用且有效。  
This benchmark proves the Codex and Claude Code adapters with two evidence layers.

- 确定性本地检查：直接验证 Gateway、MCP server 和 hook bridge。  
  Deterministic local checks validate the Gateway, MCP server, and hook bridge.
- 真实平台检查：通过 Codex 与 Claude Code 命令入口执行。  
  Real platform checks run through Codex and Claude Code command entrypoints.

对比矩阵只保留两组。  
The comparison intentionally keeps only two modes.

- `no-memory`：隔离配置，不启用 MCP server 或 hooks。  
  `no-memory`: isolated config with no MCP servers and no hooks.
- `mcp+hooks`：正式适配路径，启用 MCP 搜索工具和生命周期 hooks。  
  `mcp+hooks`: shipped adapter path with MCP search tools plus lifecycle hooks.

## 前置条件 / Prerequisites

从独立 proof repo 复跑时，先指向干净的源码 checkout。  
When rerunning from this standalone proof repo, point the harness at a clean source checkout first.

```sh
export TENCENTDB_AGENT_MEMORY_REPO=/path/to/TencentDB-Agent-Memory
export MEMORY_TENCENTDB_GATEWAY_URL=http://127.0.0.1:8420
```

如果 `TENCENTDB_AGENT_MEMORY_REPO` 未设置，脚本会假设自己仍在源码仓库中运行。  
If `TENCENTDB_AGENT_MEMORY_REPO` is not set, the script assumes it is running inside the source repository.

## 命令 / Commands

```sh
node benchmarks/adapter-memory/run.mjs smoke
node benchmarks/adapter-memory/run.mjs local --repeats 3 --limit 20
node benchmarks/adapter-memory/run.mjs agents --repeats 3 --limit 20
node benchmarks/adapter-memory/run.mjs proofs
node benchmarks/adapter-memory/run.mjs report
```

常用定向命令。  
Useful targeted commands.

```sh
node benchmarks/adapter-memory/run.mjs agents --platform codex --mode mcp+hooks --repeats 3 --limit 20
node benchmarks/adapter-memory/run.mjs agents --platform claude-code --mode no-memory --repeats 3 --limit 20
node benchmarks/adapter-memory/run.mjs local --repeats 3 --limit 20
```

## 输出 / Outputs

- `benchmarks/adapter-memory/results/` 保存本地原始评分记录和平台事件流。该目录已被 git 忽略，因为 raw logs 可能包含本地路径和诊断信息。  
  `benchmarks/adapter-memory/results/` stores local raw scoring records and platform events. It is gitignored because raw logs may include local paths and diagnostics.
- `docs/benchmark-artifacts/results/` 保存面向维护者的脱敏结果摘要，由 `npm run benchmark:adapters:report` 生成。  
  `docs/benchmark-artifacts/results/` stores maintainer-facing sanitized summaries generated by `npm run benchmark:adapters:report`.
- `docs/benchmark-artifacts/proofs/` 保存从 raw platform event streams 提炼出的 proof pages。  
  `docs/benchmark-artifacts/proofs/` stores proof pages distilled from raw platform event streams.
- `docs/benchmark-artifacts/screenshots/` 保存 proof page 的截图证据。  
  `docs/benchmark-artifacts/screenshots/` stores screenshot evidence captured from proof pages.
- `docs/adapter-benchmark-results.md` 是最终 benchmark 报告。  
  `docs/adapter-benchmark-results.md` is the final generated benchmark report.

生成报告和证明页时不能包含密钥。  
Generated reports and proof pages must not include secrets.
